## Использование

### При первом развёртывании

```bash
./deploy.sh --production
```

Скрипт автоматически:
1. Сгенерирует .env с безопасными паролями
2. Предложит отредактировать конфигурацию
3. Проверит критические параметры

### Редактирование существующей конфигурации

```bash
./manage.sh env:edit
```

Доступные опции:
1. Быстрое редактирование (домен, токены, пароли)
2. Полное редактирование в nano/vim

### Обновление с редактированием

```bash
./deploy.sh --update --backup
```

Скрипт создаст бэкап и предложит отредактировать .env перед обновлением.

## Примеры

### Быстрая настройка платежных шлюзов

```bash
$ ./manage.sh env:edit

Хотите отредактировать .env? (y/N): y

Выберите способ редактирования:
  1) Быстрое редактирование (ключевые параметры)
  2) Полное редактирование (текстовый редактор)

Ваш выбор (1-2): 1

Введите новые значения (Enter = сохранить текущее):

YOOKASSA_SHOP_ID [placeholder]: 12345
YOOKASSA_SECRET_KEY [текущее]: sk_test_abc123...
STRIPE_SECRET_KEY [текущее]: sk_test_xyz789...

Конфигурация обновлена

Перезапустить сервисы для применения изменений? (y/N): y
```

### Смена домена

```bash
$ ./manage.sh env:edit

# Выберите опцию 1
# Измените XRAY_TLS_DOMAIN

# Скрипт автоматически обновит:
# - TELEGRAM_WEBHOOK_URL
# - APP_URL
# - ADMIN_EMAIL
```

## Технические детали

### Функция `edit_env()` в deploy.sh

Расположена: строки 408-597

**Возможности:**
- Отображение таблицы текущих значений
- Интерактивный выбор режима редактирования
- Быстрое редактирование через sed
- Полное редактирование через nano/vim
- Автоматическая проверка критических параметров
- Генерация новых значений при необходимости

### Функция `cmd_env_edit()` в manage.sh

Расположена: строки 233-337

**Возможности:**
- Проверка существования .env
- Отображение текущих значений
- Два режима редактирования
- Автоматический перезапуск сервисов

### Зависимости

- `openssl` — генерация безопасных паролей
- `nano` или `vim` — текстовое редактирование
- `sed` — быстрое редактирование
- `grep` — поиск параметров
- `curl` — проверка webhook

## Известные проблемы

- ⚠️ На Windows bash недоступен (используйте WSL или Linux сервер)
- ⚠️ Если нет nano/vim, потребуется ручное редактирование
